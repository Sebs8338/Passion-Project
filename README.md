## 📌 Project Overview

Passion-Project creates a secure foundation for hosting websites on Google Cloud Platform. It sets up private servers, data storage, and traffic management systems without manual configuration. This project automates the deployment of web applications so they run safely behind a global load balancer. It uses industry standards to ensure your application stays online and responds quickly to visitors.

## 🛠 Features

*   **Private Hosting:** Servers live in a protected network environment with no direct public access.
*   **Global Traffic Management:** A load balancer directs incoming visitors to the nearest or best available server.
*   **Secure Storage:** Cloud storage artifacts save your application data in a reliable location.
*   **Infrastructure Automation:** Code defines every part of the setup to ensure consistent performance.
*   **Web Server Configuration:** Nginx manages incoming requests to provide fast content delivery.

## ⚙️ How to Use the Application

The application configuration resides in a simple menu. Open the program from your Start menu to reach the dashboard.

1. **Sign In:** Enter your Google Cloud account credentials when prompted.
2. **Select Project:** Pick your target Google Cloud project from the dropdown list.
3. **Configure Environment:** Adjust the slider for server capacity if you expect high traffic.
4. **Deploy:** Click the button labeled Build Infrastructure. The application will handle the setup of your servers and network links automatically.
5. **Monitor:** Use the dashboard to watch the progress of your deployment. Green indicators confirm successful communication with your cloud environment.

Your data stays within your controlled environment. The application acts as an interface to your cloud account. It does not store your credentials on local disks long-term. All traffic moving between your local machine and your cloud resources uses encrypted channels.

## 📈 Troubleshooting Common Issues

**Slow Loading Times**
Test your internet speed. If your connection seems slow, wait for the network to stabilize before you restart the deployment.

**Permission Denied Errors**
Ensure you signed in with a Google account that has project editor rights. You can check these permissions in your Google Cloud Console dashboard online.

**Application Will Not Open**
Try restarting your computer. If the problem persists, uninstall the application, delete the local folder, and run the installer again.

**Missing Icons or Visuals**
Update your graphics drivers if the interface appears distorted. The application relies on standard Windows interface components for all visual elements.
